# Private Reading Deployment Guide

This document describes the expected directory structure for a production deployment of Private Reading as a system service.

## Directory Structure

The Private Reading service should be installed at `/opt/private-reading/` with the following structure:

```
/opt/private-reading/
├── input/           # Input files directory
├── output/          # Raw output files directory
├── processed/       # Final processed files directory
├── config/          # Configuration files directory
├── logs/            # Log files directory
└── venv/            # Virtual environment
```

## Directory Descriptions

### `/opt/private_reading/input/`
- **Purpose**: Contains raw input files for processing (e.g., text files to convert to audio)
- **Usage**: Files placed here are picked up by the input path unit and processed by the service
- **Permissions**: Readable by the `private-reading` user

### `/opt/private_reading/output/`
- **Purpose**: Contains raw output files generated by the processing pipeline
- **Usage**: Intermediate files that may be further processed or moved to the processed directory
- **Permissions**: Writable by the `private-reading` service

### `/opt/private_reading/processed/`
- **Purpose**: Contains final processed files ready for consumption
- **Usage**: Successfully processed files that have passed all validation steps
- **Permissions**: Readable by external consumers of the output

### `/opt/private_reading/config/`
- **Purpose**: Stores configuration files for the service
- **Usage**: Custom configuration overrides, templates, and user-defined settings
- **Permissions**: Writable by the `private-reading` user

### `/opt/private_reading/logs/`
- **Purpose**: Stores log files generated by the service
- **Usage**: Application logs, error reports, and audit trails
- **Permissions**: Writable by the `private-reading` service

### `/opt/private_reading/venv/`
- **Purpose**: Python virtual environment containing dependencies
- **Usage**: Contains the `bin/activate` script and installed packages
- **Permissions**: Readable by the `private-reading` service; executable binaries

## File Permissions

The service runs under a dedicated system user `private-reading` with the following recommended permissions:

| Directory | Owner | Permissions | Purpose |
|-----------|-------|-------------|---------|
| `/opt/private-reading/` | root | 0755 | Base installation directory |
| `input/` | private-reading | 0755 | Readable input files |
| `output/` | private-reading | 0755 | Readable output files |
| `processed/` | private-reading | 0755 | Readable processed files |
| `config/` | private-reading | 0755 | Readable configuration |
| `logs/` | private-reading | 0755 | Writable log directory |
| `venv/` | private-reading | 0755 | Virtual environment |

## Environment File

The service uses an environment file at `/opt/private-reading/.env` for configuration. A template is provided at `plans/.env.example`.

### Required Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `TTS_API_ENDPOINT` | TTS service API endpoint | `https://api.example.com/v1/tts` |
| `TTS_API_KEY` | TTS service authentication key | `your-api-key-here` |

### Directory Configuration

These variables should match your installation directory structure:

| Variable | Description | Default |
|----------|-------------|---------|
| `INPUT_DIR` | Input files directory | `/opt/private_reading/input` |
| `OUTPUT_DIR` | Output files directory | `/opt/private_reading/output` |
| `PROCESSED_DIR` | Processed files directory | `/opt/private_reading/processed` |

### Logging Configuration

| Variable | Description | Values |
|----------|-------------|--------|
| `LOG_LEVEL` | Logging verbosity level | `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL` |
| `LOG_FORMAT` | Output format for logs | `json`, `text` |

### Service Configuration

| Variable | Description | Default |
|----------|-------------|---------|
| `SERVICE_USER` | User to run the service as | `private-reading` |
| `SERVICE_GROUP` | Group to run the service as | `private-reading` |

### Audio Processing

| Variable | Description | Default |
|----------|-------------|---------|
| `AUDIO_SAMPLE_RATE` | Output audio sample rate | `44100` |
| `OUTPUT_FORMAT` | Output audio format | `mp3` |

### Chunking Settings

| Variable | Description | Default |
|----------|-------------|---------|
| `CHUNK_SIZE` | Maximum chunk size in characters | `1000` |
| `CHUNK_OVERLAP` | Overlap between chunks | `50` |

## Installation Script

The installation script at [`plans/install.sh`](install.sh:1) automates the deployment process:

```bash
sudo ./plans/install.sh
```

The script performs the following steps:

1. Validates root privileges
2. Creates the required directory structure
3. Sets up the Python virtual environment
4. Installs dependencies from `requirements.txt`
5. Copies systemd service files to `/etc/systemd/system/`
6. Reloads the systemd daemon
7. Enables and starts the service

## Systemd Files

The systemd integration files are located in [`plans/systemd/`](systemd:1):

- [`private_reading.service`](systemd/private_reading.service:1) - Main service unit file
- [`private-reading-input.path`](systemd/private-reading-input.path:1) - Path unit for monitoring input directory

### Service Management Commands

```bash
# Check service status
systemctl status private_reading.service

# View logs
journalctl -u private_reading.service -f

# Restart service
sudo systemctl restart private_reading.service

# Enable on boot
sudo systemctl enable private_reading.service
```

## Troubleshooting

### Service Fails to Start

1. Check logs: `journalctl -u private_reading.service`
2. Verify directory permissions
3. Confirm environment file exists and is readable
4. Check systemd unit files syntax: `systemd-analyze verify /etc/systemd/system/private_reading.service`

### Permission Denied Errors

Ensure the `private-reading` user exists:

```bash
sudo useradd -r -s /bin/bash private-reading
```

### API Connection Issues

Verify the TTS API endpoint and key in the `.env` file are correct and accessible.
